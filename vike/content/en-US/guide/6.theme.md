# Theme System

The built-in theme system covers primary color, color mode, radius, and font customization. After extending the config, an inline init script is injected so the theme is applied before first paint, avoiding flicker on navigation or refresh (no FOUC).

## Using theme components

The framework provides two ready-made components: `ThemeToggle` (a light/dark switch button) and `ThemeSettings` (a full theme panel).

```vue
<script setup>
import { ThemeToggle } from 'vike-vue-content/components/theme-toggle'
import { ThemeSettings } from 'vike-vue-content/components/theme-settings'
</script>

<template>
  <header>
    <h1>My App</h1>
    <div>
      <ThemeSettings />
      <ThemeToggle />
    </div>
  </header>
</template>
```

> Remember to import the styles in your layout: `import 'vike-vue-content/index.css'`. This file already includes all CSS variable definitions — no need to redeclare them.

## Customizable options

| Dimension | Options |
| --- | --- |
| Primary | `red` `orange` `amber` `yellow` `lime` `green` `emerald` `teal` `cyan` `sky` `blue` `indigo` `violet` `purple` `fuchsia` `pink` `rose` (17) + Black |
| Neutral | `slate` `gray` `zinc` `neutral` `stone` (5) |
| Radius | `0` `0.125` `0.25` `0.375` `0.5` (5 presets) |
| Font | `Inter` `system-ui` `Roboto` `Open Sans` `Montserrat` `Poppins` `Outfit` `Raleway` (8) |
| Color Mode | light / dark / follow system |

The `ThemeSettings` panel also supports exporting CSS or JSON config, and a one-click reset.

## Programmatic control

Use the `useTheme()` composable to read and write the theme in code:

```vue
<script setup>
import { useTheme } from 'vike-vue-content/composables/theme'

const {
  theme,           // reactive theme config
  isDark,          // whether dark mode is active
  primary,         // primary color (read/write)
  neutral,         // neutral color (read/write)
  radius,          // radius (read/write)
  font,            // font (read/write)
  mode,            // color mode: 'light' | 'dark' | 'system'
  toggleDarkMode,  // toggle dark mode
  setBlackAsPrimary, // use black as the primary color
  resetTheme,      // reset the theme
  exportCSS,       // export a CSS string
  exportConfig,    // export a JSON config string
} = useTheme()
</script>
```

`primary`, `neutral`, `radius`, `font`, and `mode` are writable computeds — assign directly to apply:

```ts
primary.value = 'violet'
mode.value = 'dark'
radius.value = 0.5
```

The theme config shape:

```ts
interface ThemeConfig {
  colors: { primary: string; neutral: string }
  radius: number
  font: string
  colorMode: 'light' | 'dark' | 'system'
  blackAsPrimary?: boolean
}
```

## Usage tips

In layout components, just reference the variables directly — no need to write duplicate dark mode overrides:

```vue
<style scoped>
.card {
  background: var(--vvc-bg);
  color: var(--vvc-text);
  border: 1px solid var(--vvc-border);
  border-radius: var(--vvc-radius);
}

.card:hover {
  background: var(--vvc-bg-elevated);
}

/* For translucent backgrounds, use color-mix */
.header {
  background: color-mix(in srgb, var(--vvc-bg) 75%, transparent);
  backdrop-filter: blur(12px);
}
</style>
```

The framework automatically switches all variable values under the `.dark` class on the root node, so the pattern above works seamlessly in both light and dark modes.
