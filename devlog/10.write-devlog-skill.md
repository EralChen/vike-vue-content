# 2026-05-29: 沉淀 Dev Log 写作 Skill

## 目标

- 将本仓库的 Dev Log 写作流程沉淀为可复用 skill。
- 减少后续记录时对编号、索引、内容粒度和验证格式的重复判断。
- 保持 `devlog/` 面向后来者，而不是变成开发过程流水账。

## 最终改动

- 新增 `skills/write-devlog/SKILL.md`。
- Skill 明确 Dev Log 的触发场景、编号规则、文件模板、写作边界和验证步骤。
- 更新 `devlog/README.md`，把本次任务加入索引，并将下一任务示例调整为 `11.<slug>.md`。
- 修正 `devlog/9.devlog-task-files.md` 中过期的下一个编号示例。

## 关键决策

- 编号以当前 `devlog/*.md` 的真实文件集合为准，不依赖旧日志中的示例数字。
- 每次写 Dev Log 应优先记录最终事实、关键决策和验证结果。
- 已被推翻的中间方案只在能解释当前决策时简短出现，并必须明确标记为旧方案。
- `devlog/README.md` 只维护规则和索引，不承载任务正文。
