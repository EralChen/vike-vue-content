# 2026-05-28: 拆分 query 运行时入口

## 目标

- 修复运行时 API 混入 `vike-vue-content/config` 导致的入口职责问题。
- 保持 `vike-vue-content/config` 作为纯 Vike extension 入口。
- 为内容查询建立独立运行时入口：`vike-vue-content/query`。

## 最终改动

- `config/+config.ts` 回到纯 default export，只保留 Vike extension 元数据。
- 新增 `query/` 源码包，导出 `queryCollection`、`ContentEntry`、`QueryBuilder` 等 API。
- `vike/pages/index/+data.ts` 改为从 `vike-vue-content/query` 导入 `queryCollection`。
- `entry/build.sh` 纳入 `query` 构建，并从源码包 `package.json` 合并 runtime dependencies 到 `entry/package.json`。

## 关键决策

- `vike-vue-content/config` 不能暴露运行时命名导出，否则会和 Vike config-time loader 的语义混在一起。
- 运行时查询能力通过 `vike-vue-content/query` 暴露。
- `vike/` 通过 `file:../entry` 验证发布包消费路径；新增 entry 输出目录后需要刷新 file dependency 安装态。

## 验证记录

通过：

```bash
pnpm -C entry run build
pnpm --filter @vike-vue-content/vike update vike-vue-content
cd vike && node -e "import('vike-vue-content/query').then(m=>console.log(Object.keys(m)))"
pnpm -C vike run build
```

结果：`vike-vue-content/query` 可导入并提供 `queryCollection`；Vike build 通过。
