# 1. 建立 README、Roadmap、Config 与 Vike 消费端基础

## 目标

- 建立面向使用者的 README 和面向维护者的路线图。
- 新增真实的 `vike-vue-content/config` 源码包。
- 让 `vike/` 示例应用通过 `file:../entry` 消费构建后的发布包。

## 最终改动

- `README.md` 成为 npm 使用者入口，说明当前能力、安装方式、Vike 集成和相关资源。
- `ROADMAP.md` 保留在仓库根目录，用于长期产品路线，不进入 `entry/` 发布产物。
- `config/` 源码包负责生成 `vike-vue-content/config`。
- `entry/build.sh` 纳入 `config` 构建，并能为 `+config.js` 生成 `./config` 导出。
- `vike/package.json` 使用 `vike-vue-content: file:../entry` 验证发布包消费路径。
- `vike/pages/+config.ts` 在 `extends` 中接入 `vikeVueContent`。

## 关键决策

- `entry/` 是生成的 npm 发布目录，业务源码不写在 `entry/`。
- `config` 包采用 Vike extension 的 `+config` 入口约定，而不是普通 `index.es.js` 入口。
- `vike-vue-content/config` 保持纯 Vike extension default export，不承载运行时 query API。
- README 只承诺当前可用能力；长期构想写入 `ROADMAP.md`。

## 验证记录

通过：

```bash
pnpm -C entry run build
node -e "import config from './entry/config/+config.js'; console.log(JSON.stringify(config, null, 2));"
```

结果：`entry/config/+config.js` 和类型声明生成成功，默认导出为最小 extension 元数据：

```json
{
  "name": "vike-vue-content",
  "require": {
    "vike": ">=0.4.191"
  }
}
```
