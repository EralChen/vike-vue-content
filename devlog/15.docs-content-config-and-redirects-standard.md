# 15. 收敛 docs 内容配置、导航与重定向标准

## 当前标准

- docs 消费端页面保留三个稳定锚点：
  - `+config.ts` 使用 `defineDocsPageConfig({ collection, contentDir, title? })`
  - `+route.ts` 使用零参数 `createDocsRoute()` 或 `export { createDocsRoute as default } from 'vike-vue-content/docs/route'`（插件均能正确注入 base）
  - `+onBeforePrerenderStart.ts` 使用零参数 `createDocsPrerender()`
- 内容目录的 sidecar 配置统一为 `.config.yml`：
  - 顶层 `redirect`
  - `navigation.hidden`
  - `navigation.flatten`
  - `navigation.label`
- docs redirect 的最终执行层是 Vike 全局 `redirects`：
  - 内容层继续声明 redirect
  - 扩展在全局 config 阶段扫描 consumer docs 页面和 content，生成最终 `redirects`
  - 不再在 `data()` 里做 redirect 控制，也不再依赖客户端 replace 兜底作为主路径

## 关键事实

- `docs.base` 已删除，docs route base 由页面锚点位置推导，不再作为 page-local `docs` 公开配置。
- page-local 自定义 config 不能稳定地直接产出 Vike 全局 `redirects`：这会触发 Vike 的 global-config 警告，甚至命中内部 bug。最终方案改为在扩展全局 config 中扫描 consumer docs 页面和内容，再统一生成全局 redirect map。
- `queryCollectionPaths()` 只枚举真实内容页面，不再把 redirect source path 当作普通 docs 页面加入 prerender 列表。
- docs data hook 只负责内容、导航和前后页数据，不再承担 redirect 控制流。

## 验证记录

通过：

```bash
pnpm -C entry run build
pnpm -C vike run build
```

结果：

- `entry` 构建通过，`config/query/components` 等产物与类型同步到 `entry/`。
- `vike` 构建通过；docs 相关 prerender 输出恢复正常，旧的 page-local/global redirect 警告不再作为最终实现路径保留。
- 仍存在 Rolldown 对 `@vueuse/core` 的 `INVALID_ANNOTATION` 非致命警告，与本次标准收敛无关。

## 当前边界

- 在 dev/client-side navigation 下，点击一个同时又会被 docs route 命中的 redirect source URL，是否会先命中 Vike 的服务端 redirect，取决于 Vike 当前 client router 的行为；这不是内容配置格式本身的问题。
