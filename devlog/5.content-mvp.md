# 2026-05-28: 建立最小内容闭环

## 目标

- 参考 `.dev/content` 的集合与查询思想，跑通最小内容读取链路。
- 在 `vike/` 示例应用中实际读取 Markdown 内容并展示。

## 最终保留的能力

- `vike/content/docs/index.md` 作为最小内容样例。
- `vike/pages/index/+data.ts` 从内容目录读取 docs 页面数据。
- `vike/pages/index/+Page.vue` 展示 frontmatter 信息和正文渲染结果。
- 查询接口沿用 `queryCollection('docs').path('/docs').first()` 这种链式形态。

## 后续修正

- 运行时 query API 后来从 `config/` 迁移到独立的 `query/` 包，见 [6.query-entry-split.md](6.query-entry-split.md)。
- Markdown 解析和渲染后来切换为 Comark AST 与 `ContentRenderer`，见 [7.comark-ast-renderer.md](7.comark-ast-renderer.md)。
