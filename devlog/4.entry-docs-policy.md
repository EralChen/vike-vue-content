# 2026-05-28: 收敛 entry 发布文档

## 目标

- 明确 `entry/` 是 npm 发布目录，不承载源码仓库的开发文档。
- 避免把 roadmap、developer docs、devlog 等维护者上下文打进 npm 包。

## 最终改动

- `entry/build.sh` 的文档同步范围收敛为：根 `README.md` 和 `LICENSE`。
- `ROADMAP.md`、`development/`、`devlog/` 只保留在源码仓库。
- 生成产物中不保留历史调试文档或开发日志。

## 关键决策

- npm 包面向使用者，只需要 README、LICENSE 和构建产物。
- 工程维护信息放在源码仓库，不污染发布包。
- 新增文档目录时，需要同时确认 `entry/build.sh` 不会同步它。

## 验证记录

通过：

```bash
pnpm -C entry run build
find entry -name "*.md" -print | sort
cd entry && npm pack --dry-run
```

结果：`entry/` 下只有 `entry/README.md` 一个 Markdown；发布包不包含 `ROADMAP.md`、`development/` 或 `devlog/`。
