# 26. Demo 组件切换到 preview/source 数组 API

## 目标

- 废弃 `demo` 的 `name` API，统一改为 `preview`。
- 让 `source` 接收源码 key 数组，并自动渲染成 `code-group` 风格的源码面板。

## 改动文件

- `components/demo/src/ctx.ts`
- `components/demo/src/index.vue`
- `components/demo/src/types.ts`
- `vike/pages/+content.ts`
- `vike/content/en-US/3.components/demo.md`
- `vike/content/zh-CN/3.components/demo.md`

## 当前标准

### 公共写法

```md
:::demo{preview="hello", source="['hello']"}
:::
```

### 组件语义

- `preview` 选择 `demos` 中注册的实时预览组件。
- `source` 选择 `sources` 中注册的源码 key，标准形状是数组。
- MDC 静态属性写法 `source="['a', 'b']"` 会在运行时归一化成 `string[]`。
- 若未显式传 `source`，且 `sources[preview]` 存在，则默认使用同名源码 key。
- `#source` slot 仍然优先；只在没有 slot 时才使用自动源码面板。

### 自动源码面板

- 自动源码区复用 `CodeGroup + ProseCode` 结构，和手写 `#source + code-group` 保持同一套容器样式。
- 文件名和语言根据 source key / 源码内容推断；与 preview 同名的 source 默认按 `vue` 处理。

## 关键决策

- 不保留 `name -> preview` 兼容别名，直接收敛到新 API，避免文档和组件长期双写法并存。
- `vike/pages/+content.ts` 改为 `hello.vue?raw`，保证示例页里的预览组件与源码展示来自同一个文件。
- 不手改 `components/dist` 或 `entry/` 产物；这些目录继续通过构建生成。
