# 26. Demo 组件切换到 preview/source 数组 API

## 目标

- 废弃 `demo` 的 `name` API，统一改为 `preview`。
- 让 `source` 接收源码 key 数组，并自动渲染成 `code-group` 风格的源码面板。

## 改动文件

- `components/demo/src/ctx.ts`
- `components/demo/src/index.vue`
- `components/demo/src/types.ts`
- `vike/pages/+content.ts`
- `vike/content/en-US/3.components/demo.md`
- `vike/content/zh-CN/3.components/demo.md`

## 当前标准

### 公共写法

```md
:::demo{preview="hello/index.vue", :source='["hello/index.vue", "hello/data.txt"]'}
:::
```

最简写法（source 省略时自动 fallback 到 preview 对应的源码）：

```md
:::demo{preview="hello/index.vue"}
:::
```

### 组件语义

- `preview` (required, `string`): 选择 `demos` 中注册的实时预览组件。key 为 demos 目录下的相对路径（含 `.vue` 扩展名），如 `hello/index.vue`。
- `source` (optional, `string | string[]`): 选择源码 key。支持字符串或数组。未传时若 `parsedSources[preview]` 存在则默认使用。
- MDC 的 `:source` 前缀冒号表示 JS 表达式绑定，运行时值为真正的 `string[]`。

### 数据流

```
Demo 组件
  ├── useContentRenderer() → 注入 { demos, parsedSources, components }
  ├── previewKey = props.preview.trim()
  ├── demoComponent = demos.value[previewKey]
  └── sourceKeys = props.source ? normalizeSourceKeys(props.source)
                              : parsedSources[previewKey] ? [previewKey]
                              : []
```

源码区渲染：

```
sourceKeys → 遍历 → ComarkRenderer(tree=parsedSources[key], filename=getFilename(key))
                    ↓
               CodeGroup (tab 切换)
```

## 关键决策

- 不保留 `name -> preview` 兼容别名，直接收敛到新 API。
- `source` 未传时自动 fallback 到 `preview` 对应的源码，简化最常见的单文件 demo 写法。
- 文件名从 key 路径中提取（取最后一段），如 `hello/index.vue` → `index.vue`。
