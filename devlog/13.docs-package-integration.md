# 2026-05-29: 收敛 docs 包集成、预渲染与类型扩展

## 目标

- 把 docs 页面的大部分 wiring 收进 `vike-vue-content`，降低消费端样板。
- 在 Vike 0.4.x 的真实约束下，保持 `/docs/**` 的静态预渲染可用。
- 让 `docs` 自定义 config 在包侧和消费侧都有稳定类型，而不是靠本地临时类型兜底。

## 最终改动

- `config/+config.ts`：
  - 注册 `docs` 自定义 config。
  - 通过 `meta.effect` 自动挂上 `Page` 与 `data`，消费方不再为 docs 页面单独写这两项配置。
  - 用 `declare global { namespace Vike { interface Config / ConfigResolved ... } }` 扩展 `docs` 配置类型。
  - `DocsPageOptions` 改为从公开的 `config/docs/+config.ts` 导出，避免根扩展的 d.ts 指向未公开的私有类型文件。
- `config/docs/prerender/index.ts`：
  - docs 预渲染继续由包提供统一实现。
  - 读取 `globalContext.dangerouslyUseInternals._pageConfigs[*].configValues.docs.value` 获取自定义 `docs` 配置，再结合 `queryCollectionPaths()` 枚举待预渲染 URL。
  - 相对 `contentDir` 统一解析为 `path.join(process.cwd(), contentDir)`，保持 build/SSR/preview 一致。
- `vike/pages/docs/`：
  - `+config.ts` 收敛为纯 `docs` 选项对象。
  - `+route.ts` 继续直接复用 `vike-vue-content/docs/route`。
  - `+onBeforePrerenderStart.ts` 继续直接复用 `vike-vue-content/docs/prerender`。
- `vike/app.d.ts`：统一 `import 'vike-vue-content/config'`，让消费侧 TypeScript 项目显式加载包侧的 Vike namespace augmentation。
- `README.md`：更新 docs 集成说明，明确当前最小锚点是 `+config.ts`、`+route.ts` 和 `+onBeforePrerenderStart.ts`。

## 关键决策

- **Vike 的 config 类型扩展机制就是 `namespace Vike` declaration merging**：`vike-vue-content` 应按这个机制扩 `Config` / `ConfigResolved`，而不是在消费侧发明本地 `DocsConfig` 作为长期方案。
- **`Page` / `data` 与 `route` / `onBeforePrerenderStart` 的边界不同**：前两者可以由 `meta.effect` 自动挂载；后两者在 Vike 0.4.x 下仍然需要物理 runtime 文件锚点。当前三文件锚点是可验证的最小形态。
- **prerender 阶段不能从公共 `globalContext.pages` 读取自定义 page config**：公共投影不包含 `docs` 这类自定义配置，实际可用数据在内部 `_pageConfigs` 上，因此 prerender 实现必须走 `dangerouslyUseInternals`。
- **类型产物必须闭合**：根扩展的 d.ts 只能引用发布包内实际存在的公开类型路径，不能再把消费侧拉回工作区源码或未发布的私有声明文件。
