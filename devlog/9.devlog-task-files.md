# 2026-05-29: 拆分 Dev Log 为任务文件

## 目标

- 将开发记录从单个长文件拆成按任务编号的文件。
- 让后来者能按主题阅读，而不是翻一整份流水账。

## 最终改动

- `devlog/README.md` 成为开发记录索引。
- 已有任务拆成 `devlog/1.*.md` 到 `devlog/9.*.md`。
- 新任务继续使用下一个编号，例如 `devlog/12.<slug>.md`。

## 关键决策

- 每个任务文件只记录当前对后来者有价值的信息：目标、最终改动、关键决策、验证结果和后续方向。
- `devlog/README.md` 只维护索引，不写长篇正文。
- `devlog/` 只属于源码仓库，不进入 `entry/`。

## 验证记录

通过：

```bash
pnpm -C entry run build
find devlog -maxdepth 1 -type f -name "*.md" -print | sort
find entry -name "*.md" -print | sort
cd entry && npm pack --dry-run
```

结果：任务文件编号完整；`entry/` 下只有 `entry/README.md` 一个 Markdown；发布包不包含 `devlog/`。
