# 2026-05-29: 拆分开发者文档目录

## 目标

- 将开发者文档从单文件模式改为目录化管理。
- 按主题拆分工程说明，降低后来者阅读成本。
- 确认开发者文档不进入 npm 发布产物。

## 最终改动

- `development/README.md` 成为开发者文档入口。
- `development/repository.md` 说明目录职责和源码边界。
- `development/package-build.md` 说明构建链路、发布目录规则、config 包约定、导出规则和依赖合并。
- `development/content-pipeline.md` 说明 query、Comark AST、ContentRenderer 和 `.dev/content` 参考策略。
- `development/workflow.md` 说明 Vike 验证、pnpm file dependency 刷新、开发记录要求和前端验收分工。
- `README.md` 指向 `development/README.md`。

## 关键决策

- 开发者文档入口保留在 `development/README.md`，根目录不再保留 `DEVELOPMENT.md`。
- 文档按主题拆分，后续新增内容优先放入已有主题文件；确实需要新主题时再新增文件。
- `development/` 只属于源码仓库，不进入 `entry/`。

## 验证记录

通过：

```bash
pnpm -C entry run build
find entry -name "*.md" -print | sort
cd entry && npm pack --dry-run
```

结果：`entry/` 下只有 `entry/README.md` 一个 Markdown；发布包不包含 `development/`。
