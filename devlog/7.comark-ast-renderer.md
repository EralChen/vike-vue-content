# 2026-05-29: 切换到 Comark AST 与 Vue 渲染闭环

## 目标

- 使用 Comark 作为 Markdown、YAML frontmatter 和 AST 解析基础。
- 让示例页渲染真实 Markdown HTML，而不是输出原始 Markdown 文本。
- 通过组件入口暴露 Vue 渲染能力。

## 最终改动

- `query/index.ts` 使用 `comark.createParse()` 解析 Markdown。
- `ContentEntry.body` 从字符串改为可序列化的 `ComarkTree`。
- `ContentEntry.frontmatter` 和 `ContentEntry.meta` 暴露解析结果。
- `query/package.json` 使用 `comark`，不再使用 `gray-matter`。
- 新增 `components/content-renderer/`，导出 `ContentRenderer`。
- `ContentRenderer` 包装 `@comark/vue` 的 `ComarkRenderer`，并在内部提供 `<Suspense>` 边界。
- `components/vite.config.ts` 将 `@comark/vue` 和 `comark` 标记为 external，避免把大型运行时依赖打进组件产物。
- `vike/pages/index/+Page.vue` 使用 `ContentRenderer` 渲染 `docsPage.body`。

## 关键决策

- 查询层只负责内容扫描、frontmatter、AST 和元数据，不直接做 Vue 渲染。
- Vue 渲染能力作为组件库入口导出：`vike-vue-content/components/content-renderer`。
- `@comark/vue` 由发布包 dependencies 承载，组件产物保持轻量。
- `ComarkRenderer` 是 async setup 组件，必须放在 `<Suspense>` 边界内。
