# 21. 主题系统实现

## 目标

为 vike-vue-content 实现完整的主题定制系统，参考 `.dev/movk-nuxt-docs` 的 ThemePicker 设计。

## 当前标准

- 主题配置由 `composables/theme/` 统一管理，入口导出 `useTheme`、类型和常量。
- `constants.ts` 是颜色映射的唯一数据源，其他文件只引用不重复定义。
- 组件通过 CSS 变量（`--vvc-color-primary`、`--vvc-radius` 等）引用主题值，不直接使用颜色常量。
- 初始化脚本在构建时从 `constants.ts` 生成，通过 `config/+config.ts` 的 `headHtmlBegin` 注入 `<head>` 开头，防止 FOUC。
- 用户偏好持久化在 `localStorage`。

### 目录结构

```
composables/theme/
├── index.ts           # 统一导出
├── types.ts           # ThemeConfig, ColorMode, ColorValue
├── constants.ts       # 颜色映射唯一数据源
├── script.ts          # 生成初始化脚本
├── useTheme.ts        # 主题组合式函数
└── useThemeStorage.ts # localStorage 持久化

components/theme-toggle/   # 颜色模式切换（light/dark）
components/theme-settings/ # 主题设置面板（颜色、圆角、字体等）
```

### 数据流

```
constants.ts → script.ts（构建时生成）→ headHtmlBegin（运行时注入）→ useTheme（客户端应用变量）
```

## 关键决策

1. **颜色映射只在 `constants.ts` 定义**：避免多处维护导致不一致。
2. **内联脚本防 FOUC**：在 `<head>` 开头注入，浏览器解析时立即执行，页面渲染前已设好 CSS 变量和 dark 类。
3. **CSS 变量而非硬编码颜色**：组件通过 `var(--vvc-*)` 引用主题，换肤不需要改组件代码。
4. **已集成主题变量的组件**：`docs-nav`、`docs-page`、`docs-toc`、`docs-surround`。

## 验证结果

- 所有包构建通过，TypeScript 类型检查通过。
- 主题切换正常工作，无 FOUC 闪烁。
