# 21. 主题系统实现

## 目标

为 vike-vue-content 实现完整的主题定制系统，参考 `.dev/movk-nuxt-docs` 的 ThemePicker 设计。

## 架构设计

### 目录结构

```
composables/theme/
├── index.ts           # 入口文件，统一导出
├── types.ts           # 类型定义（ThemeConfig, ColorMode, ColorValue）
├── constants.ts       # 常量、预设值、颜色映射（唯一数据源）
├── script.ts          # 生成初始化脚本（防 FOUC）
├── useTheme.ts        # 主题组合式函数
└── useThemeStorage.ts # localStorage 持久化

components/theme-toggle/
├── index.ts
└── src/
    ├── index.vue      # 颜色模式切换按钮（light/dark）
    ├── ctx.ts
    └── types.ts

components/theme-settings/
├── index.ts
└── src/
    ├── index.vue      # 主题设置面板
    ├── theme-vars.css # CSS 变量定义
    ├── ctx.ts
    └── types.ts
```

### 数据流

```
constants.ts ← 唯一数据源（colorMap, neutralColorMap）
    ↓
script.ts    ← 生成初始化脚本（构建时）
    ↓
config/+config.ts ← 注入 headHtmlBegin（运行时）
    ↓
useTheme.ts  ← 应用主题变量（客户端）
```

## 实现内容

### 1. 主题配置（composables/theme）

| 配置项 | 预设值 | 说明 |
|--------|--------|------|
| primary | blue | 主色调（17 种） |
| neutral | slate | 中性色（5 种） |
| radius | 0.25 | 圆角大小 |
| font | Inter | 字体 |
| colorMode | system | 颜色模式 |
| blackAsPrimary | false | 黑色作为主色 |

### 2. 组件

**ThemeToggle** - 颜色模式切换按钮
- 点击循环：light → dark → light
- 显示 ☀️/🌙 图标

**ThemeSettings** - 主题设置面板
- Primary 颜色选择（17 种 + Black）
- Neutral 颜色选择（5 种）
- Radius 圆角选择（5 种）
- Font 字体选择（8 种）
- Color Mode 切换（3 种）
- Export 导出 CSS/配置
- Reset 重置主题

### 3. 防 FOUC 方案

在 `config/+config.ts` 中注入初始化脚本：

```typescript
import { generateThemeInitScript } from '@vike-vue-content/composables/theme'

const config = {
  headHtmlBegin: generateThemeInitScript(),
}
```

脚本原理：
1. 构建时从 `constants.ts` 生成颜色映射
2. 注入到 `<head>` 开头
3. 浏览器解析时立即执行
4. 读取 localStorage，设置 CSS 变量和 dark 类
5. 页面渲染时已经是正确主题

## 使用示例

```vue

<template>
  <!-- 简单的颜色模式切换 -->
  <ThemeToggle />
  
  <!-- 完整的主题设置面板 -->
  <ThemeSettings />
  
  <!-- 编程式控制 -->
  <button @click="setPrimaryColor('green')">绿色主题</button>
</template>
```

## 设计决策

1. **数据源统一**：颜色映射只在 `constants.ts` 中定义，其他地方引用
2. **内联脚本防 FOUC**：参考 Tailwind CSS、Nuxt UI 的方案
3. **CSS 变量**：组件通过变量引用主题，不直接使用颜色值
4. **localStorage 持久化**：用户偏好保存在浏览器中
