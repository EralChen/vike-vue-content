# 14. 删除 docs.base，收敛零参数 docs 锚点

## 目标

- 去掉放错层级的 `docs.base` API，避免让 page-local `docs` 配置承担 route 前缀职责。
- 让消费端默认使用零参数 `createDocsRoute()` 和 `createDocsPrerender()`，把 docs 页面样板压到当前 Vike 0.4.x 约束下的最小形态。
- 修好 workspace 包接线，并把 `/config` 内部实现收口到更稳定、可维护的结构。

## 最终改动

- `docs/config/options.ts`：
  - `DocsPageOptions` 只保留 `collection`、`contentDir`、`title`。
  - `base` 退回为包内派生值，不再作为公开 page config 暴露给用户。
- `docs/route/index.ts`：
  - `createDocsRoute(base?)` 继续支持显式覆盖前缀，但默认走包内推导。
  - route function 额外挂上内部 base metadata，并把 `__docsBase` 写进 route params，供 data 与 prerender 阶段复用。
- `docs/prerender/index.ts`：
  - `createDocsPrerender(routeOrBase?)` 支持直接从 docs route metadata 解析前缀。
  - 遍历 `globalContext.dangerouslyUseInternals._pageConfigs` 时按 route base 过滤，避免多个 docs 入口都把 URL 推到同一个 `/docs` 前缀下。
- `docs/plugin/plugin.ts` 与 `docs/utils/page-file.ts`：
  - 编译期插件使用 `oxc-parser` AST 分析（详见 devlog 25），把零参数 `createDocsRoute()` / `createDocsPrerender()` 重写为显式 base，同时支持 re-export 写法（`export { createDocsRoute as default } from '...'`）。
  - 抽出共享的 page-file helper，统一 route/prerender transform 与 runtime fallback 的文件系统前缀推导，去掉重复逻辑。
- `config/+config.ts`：
  - 改为注册 `docsRuntimeBasePlugin()`，名称与职责和当前实现对齐。
  - 继续通过 `meta.effect` 自动挂 `Page` 与 `data`，保留 `docs` 配置类型扩展。
- `docs/config/define.ts`：
  - `defineDocsPageConfig()` 只保留用户声明的公开 `docs` 选项，不再把内部派生字段回写到 page config。
- 消费端示例与文档：
  - `vike/pages/en-US/+route.ts`、`vike/pages/zh-CN/+route.ts` 支持零参数 `createDocsRoute()` 和 re-export 两种写法，插件均能正确注入 base。
  - 对应的 `+onBeforePrerenderStart.ts` 全部收敛为 `createDocsPrerender()`。
  - `README.md` 与 `entry/README.md` 同步为零参数写法。
- workspace 接线：
  - `entry` 纳入 workspace，`vike` 改为依赖 `workspace:*` 的 `vike-vue-content`，不再使用会破坏传递依赖解析的 `link:` 方案。

## 关键决策

- **`docs.base` 不应该放在 page-local `docs` config 中**：Vike 0.4.x 的 Route Function 运行时读不到 page-local 自定义 config，这个 API 放在那里天然错位，继续保留只会制造“能配但不生效”的假象。
- **docs 前缀属于消费端文件系统位置决定的信息**：既然 base 实际由 `pages/**/+route.ts` 与 `+onBeforePrerenderStart.ts` 的物理位置决定，就应在编译期从文件路径推导，而不是依赖 runtime stack/caller 反射。
- **runtime caller 推导只保留为 fallback**：主路径已经切到编译期 AST transform（oxc-parser），`caller.ts` 只用于非 transform 场景的兜底与默认 `/docs` 行为，不再承担核心正确性。
- **当前最小锚点仍然是三个 runtime 文件中的两个**：`Page` 与 `data` 可以由 `meta.effect` 自动挂载，但在 Vike 0.4.x 下，`route` 与 `onBeforePrerenderStart` 仍然需要页面级物理锚点；当前零参数 helper 已经是可验证的最小形态。
