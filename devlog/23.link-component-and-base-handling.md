# 23. Link 组件与 Vite base 前缀处理

## 目标

新增 `Link` 组件，让站内链接在 Vite 配置了非默认 `base`（如 `"/vike-vue-content/"`）时仍然正确跳转。

## 问题

Vite 的 `base` 选项会给所有静态资源加前缀，但不会自动处理 `<a href>` 中的路由路径。当站点部署在子路径下（如 GitHub Pages），站内绝对路径链接会丢失 base 前缀导致 404。

## 方案

`Link` 组件在运行时从 Vike 注入的 `pageContext._baseServer` 读取 base，拼接后渲染为最终的 `href`：

```vue
<Link href="/getting-started">开始</Link>
<!-- base = "/vike-vue-content/" 时渲染为 -->
<a href="/vike-vue-content/getting-started">开始</a>
```

### 关键规则

- **仅处理站内绝对路径**（以 `/` 开头）：外链、锚点、相对路径原样透传。
- **base 来自运行时而非构建时**：通过 `pageContext._baseServer` 获取，不在组件库构建时硬编码，保证组件库产物与部署路径解耦。
- **尾部斜杠归一**：`base` 末尾的 `/` 在拼接前去掉，避免双斜杠。

## 消费端接入

```vue
<script setup>
import { Link } from 'vike-vue-content/components/link'
</script>

<template>
  <Link href="/docs/getting-started">Getting Started</Link>
</template>
```

`docs-nav` 和 `docs-surround` 内部已切换为使用 `Link`。

## 关键决策

1. **运行时注入而非编译时注入**：`_baseServer` 由 Vike 在运行时写入 pageContext，组件不需要感知构建配置，保持库产物路径无关。
2. **不引入 Vue Router**：Vike 不依赖 Vue Router，Link 组件只是一个带 base 拼接的原生 `<a>` 标签，不做客户端路由拦截。
