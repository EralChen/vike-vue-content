# 27. 拆分 CodePreview 基础组件与 Demo 高层封装

## 目标

- 将原本 `demo` 的插槽式预览/源码容器抽成独立基础组件 `code-preview`。
- 让 `demo` 只负责 `preview + source[]` 这种注册式快捷 API。
- 不兼容旧的 `code-preview` API，直接以当前仓库需要的抽象为准。

## 当前标准

### CodePreview（基础容器）

- 两个插槽：`default`（预览区）和 `#source`（源码区）。
- 不解析 `preview` / `source` 属性，也不依赖 demos/sources 注册表。
- 通过 `sourceVisible` 状态控制源码区展开/收起，使用 CSS grid 动画 (`grid-template-rows: 0fr → 1fr`)。
- 工具栏按钮走 `codePreview.*` locale key，支持多语言。
- 底部浮动关闭按钮：源码展开后 sticky 在底部，可一键收起。

### Demo（高层封装）

- 组合 `CodePreview`，填充两个插槽。
- `default` 插槽：渲染 `demos[previewKey]` 获取的 Vue 组件。
- `#source` 插槽：渲染 `CodeGroup` + 多个 `ComarkRenderer`，每个对应一个源码 key 的 pre-parsed AST。
- 不再承载手写 slot 文档语义；手写预览/源码时应改用 `CodePreview`。

### 自动数据注入

```
Vite 插件 (demo-auto-collect.ts)
  ├── 扫描所有 demosDir → virtual:vvc-demos     → demosByDir[dir][key] = VueComponent
  └── 扫描所有 demosDir → virtual:vvc-demo-sources → sourcesByDir[dir][key] = rawString

服务端 (docs/data/index.ts)
  └── parseDemoSources(rawString) → ComarkTree (语法高亮 AST)

DocsPage (docs-page/src/index.vue)
  ├── contentDemos = demosByDir[demosDir]
  ├── contentSources = sourcesByDir[demosDir]
  └── contentParsedSources = docsData.parsedSources

ContentRenderer (content-renderer/src/index.vue)
  ├── provide(contentDemosKey, demos)
  ├── provide(contentSourcesKey, sources)
  ├── provide(contentParsedSourcesKey, parsedSources)
  ├── provide(contentComponentsKey, components)
  └── 注册内置组件: Demo, CodePreview, CodeGroup

Demo 组件
  └── useContentRenderer() → inject 以上所有数据
```

### 文档站接入

- `ContentRenderer` 内置注册了 `CodePreview`，文档和消费端都可以直接写 `:::code-preview`。
- `demo.md` 只保留注册式用法，手写插槽场景指向 `CodePreview` 文档。

## 关键决策

- 不兼容旧 `code-preview` API：直接围绕当前抽象重建。
- `demo` 组合 `code-preview` 而非自己维护同一套 DOM/CSS，减少重复实现。
- `code-preview` 文案独立走 `codePreview.*` locale key，不复用 `demo.*`。
- 源码统一用 `ComarkRenderer` 渲染 pre-parsed AST，不走 `ProseCode` 直出字符串，保证语法高亮一致。
