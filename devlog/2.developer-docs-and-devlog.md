# 2026-05-28: 建立工程文档与记录规范

## 目标

- 为后来维护者建立稳定的工程阅读入口。
- 明确每次开发都需要沉淀目标、改动、决策和验证结果。

## 当前有效状态

- 开发者文档入口是 [../development/README.md](../development/README.md)。
- 开发记录入口是 [README.md](README.md)。
- 每个任务写入 `devlog/<number>.<slug>.md`，并更新 [README.md](README.md) 索引。
- 根目录不再保留 `DEVELOPMENT.md` 或 `DEVLOG.md`。
- `entry/` 发布产物只同步根 `README.md` 和 `LICENSE`。

## 关键决策

- 开发文档按主题拆到 `development/`，避免单个入口文档膨胀。
- 任务记录按编号拆到 `devlog/`，避免把历史流水账堆进一个文件。
- dev 文档和 devlog 都是源码仓库上下文，不进入 npm 发布包。

## 验证记录

当前结构已经通过后续任务验证：

```bash
pnpm -C entry run build
find entry -name "*.md" -print | sort
cd entry && npm pack --dry-run
```

结果：`entry/` 只包含 `entry/README.md` 一个 Markdown；发布包不包含 `development/`、`devlog/`、`ROADMAP.md` 或历史根入口文件。

## 说明

早期曾短暂使用根目录 `DEVELOPMENT.md` / `DEVLOG.md` 作为入口，并试过把它们同步进 `entry/`。这些都是已废弃的中间方案，当前有效规则以本文件和 [../development/workflow.md](../development/workflow.md) 为准。
