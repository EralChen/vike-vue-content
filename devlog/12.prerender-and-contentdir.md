# 2026-05-29: 内容驱动路由的静态预渲染与 contentDir 解析修复

## 目标

- 修复使用本框架（vike-vue-content）的文档站点无法静态预渲染 `/docs/**` 的问题。
- 清理 Vike 模板自带的示例页，去除会拖垮构建的外部数据依赖。

## 最终改动

- 删除示例页与无用组件：`vike/pages/star-wars/`、`vike/pages/todo/`、`vike/components/Counter.vue`；同步清理 `+Layout.vue` 侧边栏链接（改为 `Welcome` + `文档`）和 `index/+Page.vue` 残留的 `Counter` import。
- 库新增枚举能力：`query/index.ts` 导出 `queryCollectionPaths(collection, options)`，返回集合内全部路由路径，用于喂给 Vike 的 `onBeforePrerenderStart()`。已重建 query → entry，产物 `entry/query/index.d.ts` / `index.es.js` 含新导出。
- 消费方新增预渲染钩子：`vike/pages/docs/+onBeforePrerenderStart.ts` 调用 `queryCollectionPaths("docs", { contentDir })` 提供待预渲染 URL 列表。
- 统一 contentDir 解析：`vike/pages/docs/+data.ts`、`vike/pages/index/+data.ts`、新建的钩子，全部从 `new URL("../../content", import.meta.url)` 改为 `path.join(process.cwd(), "content")`。

## 关键决策

- **枚举能力放库、钩子放消费方**：catch-all 路由（`+route.ts`）属于“非静态路由”，Vike 无法自动枚举要预渲染哪些 URL，这是内容框架该补的能力，所以 `queryCollectionPaths` 进库。但 `contentDir`、collection 名、docs 挂载到哪个路由是消费方决定，库无法代劳，因此 `+onBeforePrerenderStart.ts` 必须留在 app 侧。
- **contentDir 必须基于 `process.cwd()` 而非 `import.meta.url`**：原写法在 dev 跑源码时侥幸正常，但打包后 `import.meta.url` 指向 `dist/`，`../../content` 解析为不存在的 `dist/content`，触发 `walkMarkdownFiles` 内的 readdir try/catch 静默返回空数组 —— 这是预渲染拿到空列表的直接根因，且生产 SSR 会一起踩坑。构建/SSR 时 cwd 均为 `vike/` 根，故统一改用 cwd 解析。
- **star-wars 示例页是 build 失败真凶**：它在预渲染时请求 `swapi.dev`，该 API 已不可达（curl exit 35 / SSL 失败）导致 `fetch failed` → exit 1。`@vueuse/core` 的两条 `INVALID_ANNOTATION` 仅为上游包无害警告，与失败无关。

## 验证记录

通过：

```bash
cd vike && npm run build
```

结果：`EXIT=0`，预渲染从 2 个页面增至 5 个：

```
dist/client/404.html
dist/client/index.html
dist/client/docs/index.html
dist/client/docs/getting-started/index.html
dist/client/docs/guide/routing/index.html
✓ 5 HTML documents pre-rendered.
```

`Cannot pre-render page /pages/docs ...` 警告消失。

库函数独立验证（node 直接调用）：

```bash
cd vike && node --input-type=module -e "import { queryCollectionPaths } from 'vike-vue-content/query'; ..."
```

结果：返回 `['/docs', '/docs/getting-started', '/docs/guide/routing']`。

## 后续方向

1. 支持 `navigation: false` frontmatter 与 `.navigation` 目录配置，让导航/预渲染可按需排除页面。
2. 评估把 contentDir 默认解析下沉进库（约定 `<cwd>/content`），减少消费方样板。
3. 考虑为 catch-all docs 路由提供 `prerender.partial` 配置示例，兼顾静态生成与按需 SSR。
