# 2026-06-01: 建立 docs 页内导航标准

## 背景

- 文档页已经具备 TOC 元数据和标题锚点，但目录只以内嵌列表形式出现在正文上方。
- 需要补成独立的页内导航能力，对齐常见文档站右侧 `On this page` 体验。

## 当前标准

- `components/docs-page` 在桌面端使用三栏布局：左侧站点导航、中间正文、右侧本页目录。
- 本页目录直接消费 `page.toc`，不单独维护第二套 heading 数据源。
- 右侧目录样式收敛为接近 Nuxt Content docs 的细线式 `On this page` 导航，而不是卡片式目录块。
- 目录点击和 hash 变化都显式触发正文标题滚动；不依赖浏览器或路由层自动处理页内锚点跳转。
- 当前可见标题会在目录中高亮。
- 小屏幕下本页目录回落到正文上方，不固定在右侧。
- 标题元素统一设置 `scroll-margin-top`，避免锚点跳转时被顶部区域遮挡。

## 结果

- docs 页面现在提供接近 Nuxt Content 成熟样式的独立页内导航栏，而不是只在页头渲染目录列表。
- heading active 状态由浏览器端观察正文标题位置驱动，不依赖手写滚动位置常量表。
- 页内导航在当前实现中会主动执行滚动，因此不会再出现 URL hash 改变但正文位置不移动的情况。
- 该能力基于现有 TOC / anchor 机制完成，没有引入新的内容元数据格式。
