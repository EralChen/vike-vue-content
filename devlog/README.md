## 记录规则

1. 新任务接着当前最大编号继续写；清理旧条目后允许编号出现缺口，缺口不回填。
2. 每个任务文件只保留事实：当前标准、关键决策、验证结果和仍然存在的边界。
3. 已被后续任务推翻的实现方式直接删掉；只有在解释当前标准不可缺省时，才允许简短提及旧方案。
4. 本文件只维护保留下来的事实索引，不追加长篇任务正文。
