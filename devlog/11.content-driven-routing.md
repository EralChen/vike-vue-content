# 2026-05-29: 内容驱动路由闭环

## 目标

- 打通 `content/<collection>/**/*.md` 到 Vike 页面路由的端到端闭环。
- 对齐 Nuxt Content（`.dev/content`）的成熟 API 形态：导航树和上一页/下一页。
- 让文档站点具备侧边栏导航、翻页和 404 处理。

## 最终改动

- `query/index.ts`：
  - `ContentEntry` 新增 `stem`（去掉扩展名的相对路径）。
  - 新增 `queryCollectionNavigation(collection, options)`，返回按目录嵌套的 `ContentNavigationItem[]` 树。
  - 新增 `queryCollectionItemSurroundings(collection, path, options)`，基于扁平化导航树返回 `[prev, next]`。
  - 新增并导出 `ContentNavigationItem` 类型。
- `vike/pages/docs/`：新增 catch-all 路由。
  - `+route.ts`：Route Function 匹配 `/docs` 与 `/docs/**`，把 `urlPathname` 放进 `routeParams.path`。
  - `+data.ts`：用 `queryCollection().path().all()` 取当前页，并发取导航树和上一页/下一页。
  - `+Page.vue`：树形侧边栏 + 内容渲染 + 翻页 + 404 区块。
  - `DocsNav.vue`：递归渲染导航树。
- `vike/content/docs/`：新增 `getting-started.md` 和嵌套 `guide/routing.md`，验证多层级路由。
- `vike/pages/index/+Page.vue`：首页增加进入 `/docs` 的入口。
- `README.md`：补充新查询导出和「内容驱动路由」说明，并重新生成 `entry/README.md`。

## 关键决策

- API 命名对齐 Nuxt Content：`queryCollectionNavigation` / `queryCollectionItemSurroundings`，便于后续迁移和对接事实规范。
- 导航是按 `path` 段嵌套的**树**，不是扁平列表；目录中间节点用 `page: false` 占位。
- 上一页/下一页通过扁平化导航树（跳过 `page: false`）再按 `path` 取相邻项实现，复用同一份导航顺序。
- 路由匹配用 Vike Route Function 承载整个 `/docs/**`，单个页面文件即可覆盖任意层级，不为每个内容文件建页面目录。
- 内容未命中时返回页面但渲染 404 区块，由内容层而非路由层决定存在性。

## 验证记录

通过：

```bash
pnpm -C query run build
pnpm -C entry run build
pnpm --filter @vike-vue-content/vike update vike-vue-content
pnpm -C vike run build
```

结果：query、entry、Vike build 均通过；`entry/query/index.d.ts` 含新导出；Vike 产物生成 `pages_docs` 与 `DocsNav` chunk；仅有两个已知的 `@vueuse/core` `INVALID_ANNOTATION` 非致命告警。

通过：

```bash
pnpm -C vike run dev -- --host 127.0.0.1 --port 3300
curl /docs /docs/getting-started /docs/guide/routing /docs/nope
```

结果：

- `/docs` → 200，标题「快速开始」，仅有「下一页」（首项）。
- `/docs/getting-started` → 200，标题「安装与接入」，同时有上一页和下一页。
- `/docs/guide/routing` → 200，标题「内容驱动路由」，仅有「上一页」（末项），嵌套路由生效。
- `/docs/nope` → 渲染 404 区块。

surround 数量符合排序预期，证明导航顺序和相邻计算正确。
